## 概要

New API は、あなたの AI ユニバースの基盤となる、次世代の統一インフラストラクチャプラットフォームです。すべての AI プロバイダーを接続し、デジタル資産を管理し、未来を構築する力を提供します。

<Callout type="info">
  New API は、OpenAI、Anthropic、Google Gemini、DeepSeek、Midjourney、Suno
  など、30 以上のグローバル AI
  サービスプロバイダーとシームレスに統合しています。
</Callout>

### 主要機能

- **統一インターフェース**：すべての AI サービスに対応する単一の API エンドポイント、OpenAI 互換形式
- **インテリジェントルーティング**：マルチチャネル負荷分散、自動フェイルオーバー、加重分散
- **柔軟な課金**：リクエスト/使用量ベースの課金、プリペイドチャージ、設定可能な倍率
- **セキュリティ管理**：トークン権限管理、モデルアクセス制御、API 呼び出し監査
- **データインサイト**：リアルタイムダッシュボード、使用統計、コスト分析
- **マルチテナントアーキテクチャ**：個人開発者、チームコラボレーション、エンタープライズデプロイメントに最適

### オープンソースライセンス（AGPLv3）

AGPLv3の下では、New APIを個人およびオープンソースプロジェクトのために自由に利用、変更、配布することができます。

**主な条項：**

- 自由に利用、変更、配布可能
- 完全なライセンステキスト：[GNU AGPLv3](https://www.gnu.org/licenses/agpl-3.0.html)
- **中核的義務**：New APIを変更し、ウェブサービス（SaaS）としてデプロイする場合、AGPLv3の下で完全なソースコードを提供する必要があります
- **ブランド要件**：すべての元のブランド、ロゴ、著作権表示を保持する必要があります

<Callout type="error">
  AGPLv3ライセンスのみを使用する場合、New
  APIのブランド識別子を削除または変更することはできません。ホワイトラベル化には商用ライセンスが必要です。
</Callout>

### 商用ライセンス

以下のシナリオでは、商用ライセンスを**取得する必要があります**：

**シナリオ1：ブランドおよび著作権情報の削除**

- 製品またはサービスからNew APIのLOGO、UIインターフェースの著作権表示、またはその他のブランド識別子を削除したい場合。

**シナリオ2：AGPLv3オープンソース義務の回避**

- New APIを変更し、ネットワーク経由でサービス（SaaS）を提供したいが、変更したソースコードをサービスユーザーに**開示したくない**場合。
- New APIを統合したソフトウェア製品を配布したいが、AGPLv3ライセンスの下で製品をリリースしたり、ソースコードを開示したくない場合。

**シナリオ3：企業ポリシーおよび統合要件**

- 所属会社のポリシー、顧客契約、またはプロジェクト要件がAGPLv3ライセンスソフトウェアの使用を許可していない場合。
- OEM統合を実行し、New APIをクローズドソースの商業製品の一部として再配布する必要がある場合。

**シナリオ4：商用サポートおよび保証が必要な場合**

- AGPLv3では提供されない商用保証（例：公式の技術サポート）が必要な場合。

**商用ライセンスの取得：**

商用ライセンスに関するお問い合わせは、[support@quantumnous.com](mailto:support@quantumnous.com)までメールでお送りください。

New APIへのすべての貢献はAGPLv3の下でライセンスされ、オープンソース版および商用版の両方に含まれる可能性があります。
